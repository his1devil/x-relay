Security model

The relay is a pipe,
not a party.

Everything that matters is sealed before it leaves your device. The server’s only job is to move opaque bytes between two peers that already trust each other.

ROADMAP → wss/TLS at the edge · Noise IK handshake
(forward secrecy) · in-payload sequence numbers

✓

End-to-end AES-256-GCM

Random-IV, authenticated. Keys live only on your Mac and phone, exchanged via QR.

✓

Blind relay

Routes ciphertext by room id. No accounts, no logs of content — it can’t read what it can’t decrypt.

✓

Abuse-hardened

Per-IP connection caps, join throttling, 8 MB frame limit, token-bucket rate limiting, replay dedup.

✓

Attachments verified

Files land only after a SHA-256 match; names are sanitized against path traversal.

FAQ

Questions, answered straight.

Does my code touch your servers? +

No. Code stays on your Mac. Transcripts stream phone-ward as ciphertext through a blind relay; the relay stores nothing and can decrypt nothing.

Which agents work? +

Claude Code and Codex natively today, plus TIL. Anything that runs in a terminal can live in a vibTTY pane; deep session mirroring currently targets Claude-family transcripts.

What happens when my Mac sleeps or the network drops? +

HALX reconnects automatically with backoff and path monitoring. Sessions resume where they were — the terminal is the source of truth, so nothing is lost in between.

How much does it cost? +

It’s a personal project — free. Public downloads are being prepared; vibTTY ships notarized, HALX via TestFlight.
